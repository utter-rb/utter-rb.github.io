Philosophy

Building a Microservice often requires the utilization of knowledge from a number of different domains. Utter's Domain-specific Microservices are source code artifacts that focus on a specific core domain, while incorporating knowledge from other supporting, and generic domains, via extensions. Utter argues that a diversity of Domain-specific Microservices will positively influence the construction of software applications. Hence, Utter intend to support microservice developers, via providing a generative approach for building Domain-specific Microservices that supports the following tents:
Everything is a Component
Utter sees services, domains, domain-extenstions as system components of your microservices.
Everything is Contextual
Utter generates context-aware system components, e.g. standalone domain-extenstions vs domain embedded extenstions.
Everything is Usable
Utter guides you into dividing your system into fine-grained components, making sure that every system component you develop is simple enough to be understood by others, hence usable if it is meant to be shared.
Everything is Reusable
Utter components are versioned Inheritable Ruby Classes for smooth scoped-evolution.
Everything is Packageable
Utter makes sure that every system component developed, is one command away form being packageable as a gem for extended reuse, among your team or the community as a whole.
Everything is Independent
Each Domain Extenstion has its own data presistance mechanism, so in one domain you might have multiple memory and file-based databases
